1. Module Purpose
The Audit & Compliance System (Module 2) provides the Compliance team with complete visibility into schedule adherence across all departments. The module combines a live attendance audit grid with a shift rules management interface, enabling auditors to detect violations, investigate attendance patterns, and enforce company scheduling policies. Managers have read-only access to audit data, while Compliance team members can fully manage shift rules.
2. Module Screens
Screen 1 — Audit Schedule Grid
The primary view displays a weekly grid with employees as rows and days of the week as columns. Each cell shows the employee's attendance status for that day, color-coded for instant recognition. Auditors can navigate between weeks, filter by department or rule, and click any cell to open a detailed attendance modal showing planned versus actual times.
Screen 2 — Shifts Management
The Shifts tab lists all defined company shift types with their name, start time, end time, and auto-calculated duration. Overnight shifts (those crossing midnight) are automatically flagged with a 🌙 label. Compliance administrators can add new shifts, edit existing ones, or delete them with a confirmation step.

4. Attendance Status Color Reference
The audit grid uses four color codes to communicate attendance status at a glance:
•	🟢 Green — On Time — Employee clocked in and out within the scheduled shift window.
•	🟠 Orange — Late Arrival — Employee clocked in after the scheduled start time (grace period: 5 minutes).
•	🔴 Red — Absent — No clock-in recorded; employee did not attend the scheduled shift.
•	🟣 Purple — Left Early — Employee clocked out before the scheduled end time (threshold: 15 minutes early).
•	🟣🟠 Combined — Late + Left Early — Employee was both late to arrive AND left before end time. Note shows exact minutes for each: e.g. "Late 6m, left 30m early".

2.  Module Overview
The Manager Schedule System allows managers (Level 2 and above) to build and manage the weekly work schedule for all employees in their department. The manager sees a grid with employees in rows and the days of the week across the top, and fills in each shift by clicking a cell.

The system supports the following key workflows:

•	Building a weekly schedule by assigning one of 6 fixed company-wide shifts to each employee
•	Copying the previous week's schedule as a starting point to save time
•	Assigning shifts in bulk to multiple employees or multiple days at once
•	Marking shifts as cover shifts and recording which employee is being covered
•	Adding optional comments to any shift assignment for context or notes
•	Viewing the full change history for any shift: who changed it, when, and why
•	Publishing the schedule when complete so audit can see it
•	Automated weekly email reminders to ensure schedules are always created on time
•	Exporting the weekly schedule as a CSV file for offline use or reporting

The system is available as a web application used by managers and HR administrators on desktop or laptop computers.

Each cell represents one employee on one day. The manager clicks a cell to assign or change a shift. Filled cells show the shift name, hours, and a color. Empty cells appear as a dashed box with a + sign, indicating no shift has been assigned yet.

The 3 Off Types

5.  Bulk Assignment - How It Works
The bulk assignment feature lets a manager assign shifts to many cells at once instead of clicking each cell individually. This is especially useful at the start of a new week when many employees share the same shift.

There are two bulk modes:

•	By employees (rows) - The manager ticks the checkbox next to multiple employee names, picks one shift, and that shift is applied to all selected employees for the selected day.
•	By days (columns) - The manager ticks the checkbox on multiple day headers, picks one shift, and that shift is applied to the selected employee for all chosen days.

When bulk mode is active, a blue information box appears on screen to guide the manager through the steps. This makes the feature easy to understand even for managers using it for the first time.

6.  Automated Email Reminders
To ensure the weekly schedule is always created on time, the system includes an automated daily check that runs at 12:00 PM every day. This works in the background without any action needed from the manager.

How the automated reminders work
The system checks every day at 12:00 PM (noon). Here is what happens:
Monday  -->  The system sends a friendly reminder email to the manager: "Please create this week's schedule."
Tuesday  -->  If the schedule is still not created, the system sends a second reminder email AND automatically copies last week's schedule into the current week.
Wednesday onward  -->  The copied schedule is already in place. No more emails are sent. The manager can still edit any shift at any time.

This automation ensures that employees are never left without a published schedule. Even if the manager forgets, a schedule is always in place by Tuesday at the latest.
